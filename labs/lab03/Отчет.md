# Отчет по лабораторной работе №3
## Контейнеризация вычислений и создание клиент-серверного приложения

**Студент**: Емельянов Матвей

**Вариант**: №7

**Стек**: Docker, Docker Compose, FastAPI, Python 3.12, Poetry

### 1. Постановка задачи
Целью работы является объединение результатов предыдущих лабораторных работ (№1 и №2) в единую систему с использованием технологий контейнеризации.
Основные задачи:
1.  Написать `Dockerfile` для изоляции окружения первой лабораторной работы.
2.  Написать `Dockerfile` для второй лабораторной работы, обеспечив компиляцию C++ модуля внутри контейнера.
3.  Реализовать веб-сервис (FastAPI) для отображения результатов.
4.  Организовать взаимодействие контейнеров и запуск системы через `docker-compose`.

### 2. Архитектура решения

Была разработана микросервисная архитектура, состоящая из трех контейнеров, объединенных общей сетью и общим томом (Volume) для обмена данными:

1.  **`lab01_container`**:
    *   Клонирует репозиторий.
    *   Устанавливает зависимости через Poetry.
    *   Производит расчеты (Python vs NumPy) и сохраняет вывод в общий файл `lab01_output.txt`.

2.  **`lab02_container`**:
    *   Содержит компилятор `g++` и инструменты сборки (`build-essential`).
    *   Компилирует модуль `pybind11` "на лету" внутри контейнера.
    *   Производит расчеты (Numba vs C++) и сохраняет вывод в общий файл `lab02_output.txt`.

3.  **`web_client`**:
    *   Веб-сервер на базе **FastAPI**.
    *   Читает текстовые отчеты из подключенного тома.
    *   Генерирует HTML-страницу с автообновлением, позволяя пользователю видеть результаты работы всех алгоритмов в браузере.

### 3. Особенности реализации

*   **Управление зависимостями**: Для уменьшения размера образов использован `python:3.12-slim` и очистка кэша `apt`.
*   **Сборка C++**: Во втором контейнере реализован pipeline сборки расширения: установка зависимостей -> запуск `build.py build_ext --inplace` -> выполнение скрипта. Это гарантирует, что бинарный файл `.so` будет совместим с архитектурой контейнера (Linux).
*   **Синхронизация**: Использован механизм `volumes` в Docker Compose для передачи результатов вычислений от воркеров к веб-серверу в реальном времени.

### 4. Инструменты разработки

*   **Docker**: Для создания изолированных воспроизводимых сред.
*   **Docker Compose**: Для оркестрации запуска трех контейнеров одной командой.
*   **FastAPI**: Для быстрой реализации веб-интерфейса.

### 5. Заключение

В ходе работы была решена проблема "works on my machine". Теперь вычислительные модули, написанные на Python и C++, гарантированно запускаются в любой среде, где установлен Docker, не требуя от пользователя ручной установки компиляторов или библиотек. 

Веб-интерфейс предоставляет удобный способ агрегации отчетов, демонстрируя преимущество клиент-серверного подхода в высокопроизводительных вычислениях.
